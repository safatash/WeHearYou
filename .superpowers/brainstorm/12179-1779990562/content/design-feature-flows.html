Design · Section 2 of 4 — Feature flows

How each of the four features works end-to-end, from user action to Google.

💬 REVIEW REPLY PUBLISHING
Reviews Inbox
→
AI drafts reply (existing)
→
Publish to Google ↗
→
Review.replyPublishedAt set
Manual publish
User edits draft → clicks "Publish to Google" → server action calls accounts.locations.reviews.updateReply → stores replyGbpId + timestamp
Auto-publish (new automation step)
New PUBLISH_GBP_REPLY step type in automation engine. Triggered after a review is synced. Publishes the AI draft directly — no human click needed.
📝 GBP POST SCHEDULER
GBP Manager → Posts
→
Write content + pick type
→
Set schedule or "Publish now"
→
Cron publishes ↗
Post types: What's New · Offer (with coupon code + expiry) · Event (with start/end date + title). Calls accounts.locations.localPosts.create. Stores gbpPostId for later edit/delete.
Image support: Optional image upload → stored to Vercel Blob → URL passed to GBP API.
🖼 PHOTO MANAGEMENT
Upload flow
File picker → upload to Vercel Blob → create GbpPhoto (DRAFT) → pick category → schedule or publish now → cron calls accounts.locations.media.create
View live photos
Gallery tab fetches all published GbpPhotos from DB (we stored them). Shows which are live on Google. Delete button calls media.delete + marks record deleted.
❓ Q&A MANAGEMENT
Nightly sync pulls questions
→
Unanswered ones surface in GBP Manager
→
User answers → pushed to Google
New question alert: When the nightly sync finds a question with no answer and syncedAt is within the last 24h, send a team notification email (same sendTeamNotificationEmail function already used by automation). No separate webhook needed.
GBP API surface used: reviews.updateReply localPosts.create/patch/delete media.create/delete/list questions.list answers.upsert — all under the existing Google OAuth connection.
