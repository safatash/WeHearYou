Design · Section 4 of 4 — Error handling & testing

How failures surface to users, and what we verify before shipping.

ERROR HANDLING
GBP API call fails (reply, post, photo)
Server action returns error message → flash banner on page. GbpPost/GbpPhoto status set to FAILED with error stored. User can retry from the UI.
Scheduled item fails in cron
Status → FAILED. GbpSyncLog records the error. Failed items appear in the UI with a "Retry" button. Cron does NOT retry automatically — avoids duplicate posts if API is flaky.
Google token expired
gbp-api.ts checks for 401 → attempts token refresh using existing GoogleAccountConnection refresh token. If refresh fails, surfaces "Reconnect Google" prompt in the GBP Manager hub.
Location not connected to GBP
GBP Manager hub shows an empty state with a "Connect Google Business Profile" CTA — links to existing Google OAuth flow (already built for review sync).
No silent failures. Every GBP write that fails must be visible somewhere — either in the UI (flash + status badge) or in GbpSyncLog. The cron endpoint returns a JSON summary that Vercel logs capture.
TESTING PLAN
Unit tests — gbp-api.ts & gbp-scheduler.ts
Mock fetch → verify correct GBP API endpoints and payloads are called.
Test scheduler: items past scheduledAt get published; future items are skipped; FAILED items are not retried.
Test token refresh: 401 response triggers refresh; second 401 surfaces error.
Integration tests — server actions
publishReply: review with draft → action succeeds → Review.replyPublishedAt set.
createPost: valid payload → GbpPost created with SCHEDULED status.
answerQuestion: GbpQuestion.answeredAt set after successful API call.
Manual smoke tests before deploy
✓ Reply draft → Publish to Google → replyPublishedAt appears in Reviews Inbox
✓ Create a "What's New" post → schedule 2 min in future → wait → verify it appears on GBP
✓ Upload a photo → publish now → verify it appears in Google Maps
✓ Answer a Q&A question → verify it appears on listing
✓ Disconnect/expire Google token → verify "Reconnect" prompt appears
