How should the new widget creation flow work?

3 approaches — each trades scope vs. experience

A

Enhanced Modal (2 steps)

Keep the "Create Widget" button → modal, but make the modal multi-step: Step 1 — visual layout gallery inside the modal (big preview cards). Step 2 — name + location. Creates a widget and redirects to the customizer.

Pros

  • Minimal page changes
  • Fast to build
  • Familiar flow

Cons

  • Cramped for many variants
  • No deep URL / back button
B

Dedicated Page — Full Trustindex Clone

New route /widgets/new. Full-page layout selector with layout-type tabs (Slider, Grid, List, Badge, Floating…), multiple visual variants per type (List I, List II…), and a top source filter. Selecting a variant → name/location form → create.

Pros

  • Rich, pro-feeling experience
  • Scales to many variants
  • Deep-linkable

Cons

  • Bigger build
  • Need real preview thumbnails
C

Dedicated Page — Simplified (Recommended)

New route /widgets/new with a visual layout gallery (one card per layout type, rendered HTML preview — no variants for now). Layout type tabs to filter. No source filter (WeHearYou is Google-only). Select a card → bottom panel slides up for name + location → create.

Pros

  • Looks great, not overwhelming
  • Right scope for current app
  • Extendable to variants later

Cons

  • Not a 1:1 Trustindex clone
