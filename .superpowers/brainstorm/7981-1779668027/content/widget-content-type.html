Widget content type — what gets displayed

When creating a widget at /widgets/new, the user picks a content type and a layout. These are independent choices.

Step 1 — What content to show?
⭐ Text Reviews
Google reviews, synced automatically. Star ratings + written text.
🎥 Video Testimonials
Published video testimonials. Each card shows a thumbnail + play button.
⭐🎥 Both — Text + Video
Mix of text reviews and video testimonials in a single embed. Videos are shown with thumbnails alongside written reviews.

Architecture with "Both" mode:
  • A single ReviewWidget gets a contentType field: TEXT | VIDEO | MIXED
  • The public API at /api/public/widgets/[token] returns reviews and/or video testimonials based on contentType
  • The embed script renders video cards (thumbnail + name + duration + play) or text cards (stars + body) or both — same layout engine handles either
  • In MIXED mode, videos are sorted together with text reviews by date
  • DB: one new column — contentType on ReviewWidget, defaulting to TEXT (fully backwards compatible)
